01

Logical Operators

FUNDAMENTALS

What are they?

Logical operators (also called connectives) are symbols that connect or modify propositions to form compound statements. They are the building blocks of all logic.

¬
Negation (NOT)
Flips the truth value of a proposition. If p is true, ¬p is false, and vice versa.
"NOT p"
¬T = F  |  ¬F = T
Example

p = "It is raining."
¬p = "It is not raining."

∧
Conjunction (AND)
True ONLY when BOTH propositions are true. Any false input makes the whole thing false.
"p AND q"
T ∧ T = T  |  T ∧ F = F
Example

p = "I studied."   q = "I slept."
p ∧ q = "I studied and I slept." (Both must be true)

∨
Disjunction (OR)
True when AT LEAST ONE proposition is true. Only false when both are false. This is inclusive OR.
"p OR q"
F ∨ F = F  |  T ∨ F = T
Example

p = "I'll eat rice."   q = "I'll eat bread."
p ∨ q = "I'll eat rice or bread (or both)."

⊕
Exclusive OR (XOR)
True when exactly ONE of the two propositions is true — NOT both. Used a lot in circuits and cryptography.
"p XOR q"
T ⊕ T = F  |  T ⊕ F = T
Example

p = "Door opens in."   q = "Door opens out."
p ⊕ q = True (it can only do one, not both)

→
Implication (IF-THEN)
False ONLY when the hypothesis (p) is true but the conclusion (q) is false. "If p, then q." A false premise makes the whole thing vacuously true.
"p implies q" / "if p then q"
F → anything = T  |  T → F = F
Example

p = "I pass the exam."   q = "I celebrate."
p → q: Only false if I pass but don't celebrate.

↔
Biconditional (IFF)
True when BOTH propositions have the same truth value. "p if and only if q." Both true or both false = True.
"p if and only if q"
T ↔ T = T  |  T ↔ F = F
Example

p = "I study."   q = "I pass."
p ↔ q: "I study if and only if I pass."
True when both study+pass, or both not-study+not-pass.

Operator Precedence (Highest → Lowest)

¬ > ∧ > ∨ > → > ↔

Just like PEMDAS — evaluate in this order when no parentheses are given. ¬ binds tightest.

02

Logical Equivalences

LAWS

What is it?

Two propositions are logically equivalent (written p ≡ q) if they have the same truth value in every possible scenario — their truth tables are identical. Equivalences let you simplify and transform logical expressions.

Key Symbol

p ≡ q means: "p and q are always equal in truth value."
Different from ↔ (biconditional is a connective; ≡ is a relation between formulas).

Identity Laws
Identity (AND)p ∧ T ≡ p
Identity (OR)p ∨ F ≡ p
Domination Laws
Domination (OR)p ∨ T ≡ T
Domination (AND)p ∧ F ≡ F
Idempotent Laws
Idempotent (OR)p ∨ p ≡ p
Idempotent (AND)p ∧ p ≡ p
Double Negation
Double Negation¬(¬p) ≡ p
De Morgan's Laws IMPORTANT
De Morgan 1¬(p ∧ q) ≡ ¬p ∨ ¬q
De Morgan 2¬(p ∨ q) ≡ ¬p ∧ ¬q
De Morgan's Example

"It is NOT the case that (I'm hungry AND tired)"
≡ "I'm NOT hungry OR I'm NOT tired"

¬(H ∧ T) ≡ ¬H ∨ ¬T

Commutative Laws
Commutative (OR)p ∨ q ≡ q ∨ p
Commutative (AND)p ∧ q ≡ q ∧ p
Associative Laws
Associative (OR)(p ∨ q) ∨ r ≡ p ∨ (q ∨ r)
Associative (AND)(p ∧ q) ∧ r ≡ p ∧ (q ∧ r)
Distributive Laws
Distributive (AND over OR)p ∧ (q ∨ r) ≡ (p ∧ q) ∨ (p ∧ r)
Distributive (OR over AND)p ∨ (q ∧ r) ≡ (p ∨ q) ∧ (p ∨ r)
Absorption Laws
Absorption (OR)p ∨ (p ∧ q) ≡ p
Absorption (AND)p ∧ (p ∨ q) ≡ p
Negation Laws
Negation (OR)p ∨ ¬p ≡ T  (Tautology)
Negation (AND)p ∧ ¬p ≡ F  (Contradiction)
Implication Equivalences
Implicationp → q ≡ ¬p ∨ q
Contrapositivep → q ≡ ¬q → ¬p
Biconditionalp ↔ q ≡ (p → q) ∧ (q → p)

Logic & Bit Operations

APPLIED

What are they?

Bit operations apply logical operators directly to binary digits (bits): 0 = False, 1 = True. Computers literally use these to process everything. Every value in memory is bits, and operations on them are just logic.

AND ( & )

Both bits must be 1. Used to mask bits — isolate specific bits.

1010
AND 1100
———
1000
OR ( | )

At least one bit must be 1. Used to set bits.

1010
OR 1100
———
1110
XOR ( ^ )

Exactly one bit is 1. Used to toggle bits and simple encryption.

1010
XOR 1100
———
0110
NOT ( ~ )

Flips every bit. 0→1, 1→0. Also called bitwise complement.

NOT 1010
———
    0101

Bit Strings

A bit string is a sequence of bits. Operations are applied bitwise — position by position.

Example — Bit String Operations

A = 0110 1010   B = 1011 0101

A AND B = 0010 0000
A OR B  = 1111 1111
A XOR B = 1101 1111
NOT A   = 1001 0101

Real-World Use

Masking: n & 0xFF extracts the lowest 8 bits (used in networking, image processing).

Toggle a flag: n ^ (1 << k) flips the k-th bit without touching others.

Check if even: n & 1 == 0 (if last bit is 0, it's even).

Logic Gates Connection

Every gate in a digital circuit IS one of these operators:

AND Gate = ∧ OR Gate = ∨ NOT Gate = ¬ XOR Gate = ⊕ NAND = ¬(∧) NOR = ¬(∨)

Propositional Logic

CORE

What is it?

Propositional logic is a formal system for reasoning using propositions — statements that are either true or false, never both, never neither. It's the foundation of all mathematical proofs, circuit design, and programming logic.

Proposition

A declarative statement with a definite truth value.

Valid Propositions

"Manila is in the Philippines." → T
"2 + 2 = 5" → F
"The sky is green." → F

NOT a Proposition

Questions, commands, opinions, or paradoxes.

Invalid Propositions

"What time is it?" → ❌ (question)
"Close the door." → ❌ (command)
"This statement is false." → ❌ (paradox)

Compound Propositions

Formed by combining simple propositions with logical operators. The truth value of a compound proposition depends entirely on the truth values of its parts and the operators used.

p = "It rains."   q = "I bring umbrella."
Compound: p → q   ("If it rains, I bring umbrella.")

Tautology

Always TRUE regardless of variable values.

p ∨ ¬p   (always true)

Aka: "Law of Excluded Middle"

Contradiction

Always FALSE regardless of variable values.

p ∧ ¬p   (always false)

Aka: "Law of Non-Contradiction"

Contingency

Sometimes true, sometimes false — depends on values.

p ∧ q   (depends on p and q)

Most regular compound propositions.

Implication Variants EXAM FAVORITE

Given: p → q ("If p, then q")

Original p → q
Converse q → p   (swap p and q)
Inverse ¬p → ¬q   (negate both)
Contrapositive ¬q → ¬p   (swap AND negate)
Example

Original: "If I study, then I pass." (p → q)
Converse: "If I pass, then I studied." (q → p)
Inverse: "If I don't study, I don't pass." (¬p → ¬q)
Contrapositive: "If I don't pass, I didn't study." (¬q → ¬p)

Key fact: Original ≡ Contrapositive   |   Converse ≡ Inverse

Arguments & Validity

An argument is a sequence of propositions (premises) followed by a conclusion. It's valid if the conclusion MUST be true whenever all premises are true.

Modus Ponens:   p,   p → q   ∴ q
Modus Ponens Example

Premise 1: "It is raining." (p = T)
Premise 2: "If it rains, I stay home." (p → q)
Conclusion: "I stay home." (q) ✓

Modus Tollens:   ¬q,   p → q   ∴ ¬p

Exam Tips

• n variables = 2ⁿ rows. Always.

• Use a pattern for filling: alternate T/F every 1 row for last var, every 2 for second-to-last, etc.

• Implication is the trickiest — remember: only False when hypothesis is T and conclusion is F.

• To prove equivalence: build both columns, check they're identical.
